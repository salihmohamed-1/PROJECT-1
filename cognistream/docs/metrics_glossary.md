# 📊 Metrics Glossary

This document explains the productivity metrics used in the CogniStream platform.

---

## Total Commits

**Description:**
Number of Git commits made by a developer during a selected time period.

**Purpose:**
Measures coding activity and development progress.

---

## Pull Requests

**Description:**
Number of pull requests opened or merged.

**Purpose:**
Tracks collaboration and code review activity.

---

## Slack Activity

**Description:**
Number of Slack messages or interactions.

**Purpose:**
Measures communication and team collaboration.

---

## IDE Activity

**Description:**
Time spent actively coding inside the IDE.

**Purpose:**
Represents actual development effort.

---

## Flow Score

**Description:**
A productivity score based on continuous coding duration.

| Coding Duration | Flow Score |
|-----------------|------------|
| Less than 45 min | Low |
| 45–89 min | Medium |
| 90 min or more | High |

---

## Event Cleaning

**Description:**
Process of removing incomplete or invalid events before storage.

**Purpose:**
Improves data quality and reporting accuracy.

---
